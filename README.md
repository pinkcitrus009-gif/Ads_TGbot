# D&D Master — Telegram-бот

Телеграм-бот, который выступает в роли Мастера подземелий (Dungeon Master) для игры по D&D 2024. Все ответы — на русском языке.

## Возможности

- `/start` — начать новую игру.
- `/reset` — сбросить историю и начать заново.
- `/status` — показать лист персонажа.
- Любое текстовое сообщение продолжает приключение.

## Установка

1. Клонируйте репозиторий и перейдите в каталог проекта.
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Скопируйте `.env.example` в `.env` и заполните значения:
   ```bash
   cp .env.example .env
   ```
5. Запустите бота:
   ```bash
   python bot.py
   ```

## Переменные окружения

Все секреты задаются ТОЛЬКО через переменные окружения (или локальный файл `.env`). В коде их хранить запрещено.

| Переменная | Назначение |
| --- | --- |
| `TG_TOKEN` | Токен Telegram-бота от @BotFather. |
| `OPENROUTER_API_KEY` | Ключ API OpenRouter. |
| `OPENROUTER_MODEL` | Модель OpenRouter. По умолчанию `openrouter/owl-alpha`. |
| `DATABASE_URL` | Строка подключения к PostgreSQL (асинхронный драйвер). |

## Хранилище данных

Постоянное состояние (персонажи, история, прогресс) хранится в **PostgreSQL**. Подключение настраивается через единственную переменную `DATABASE_URL` и использует асинхронный доступ (SQLAlchemy 2.0 + `asyncpg`):

```
postgresql+asyncpg://user:password@host:5432/dbname
```

Важно: `DATABASE_URL` должна указывать на **персистентную** базу данных. Файловые системы эфемерных хостингов (Heroku, Railway worker и т. п.) не сохраняют данные между перезапусками — поэтому локальный SQLite-файл на таком хостинге не подходит для продакшена, и нужна именно внешняя PostgreSQL.

> Примечание: интеграция с базой данных подключается на следующем этапе разработки (Phase 1). На текущем этапе заложены конфигурация и зависимости.
