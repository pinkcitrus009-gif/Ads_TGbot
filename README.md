## Обзор

D&D Master Bot — это AI Dungeon Master прямо в Telegram. Он:

- **Ведёт нарратив** — рассказывает историю, описывает мир, разыгрывает NPC
- **Следит за сюжетом** — помнит что произошло, кто где находится, что уже сделано
- **Автоматически обновляет персонажа** — когда DM говорит «ты получаешь 50 золотых», бот сам добавляет их в лист
- **Работает в группах** — 2–10 игроков в одном чате, у каждого свой персонаж
- **Управляет боем** — инициатива, порядок ходов, HP врага, урон, побег
- **Хранит всё в SQLite** — персонажи, история, квесты, сохранения сохраняются между сессиями

## Архитектура проекта

```
bot_v3/                      ← Текущая версия (используй её)
├── bot.py                   ← Главный файл: все обработчики, AI-вызовы, логика
├── database.py              ← SQLite-обёртка: все CRUD-операции
├── character.py             ← Игровая механика: расы, классы, кубики, форматирование
├── keyboards.py             ← Inline-клавиатуры Telegram
├── requirements.txt         ← python-telegram-bot + aiohttp
└── Procfile                 ← Точка входа для Railway

bot_v2/                      ← Предыдущая версия (одиночная игра)
bot/                         ← Первая версия (базовая)
```

### Технологический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.11+ |
| Telegram API | python-telegram-bot 22.8 |
| HTTP-клиент | aiohttp 3.12 |
| AI | OpenRouter API (модель configurable) |
| База данных | SQLite (через stdlib sqlite3) |
| Деплой | Railway (worker dyno) |
| Хранение данных | Railway Volume (persistent disk) |

### Поток данных

```
Игрок → Telegram → bot.py (handle_message)
                        ↓
              db.add_message (история)
                        ↓
              compress_history (если > 40 сообщений)
                        ↓
              ask_ai (OpenRouter API)
                        ↓
              parse_ai_state → apply_state_update
              (парсинг [STATE] блока, обновление БД)
                        ↓
              send reply + keyboards → Игрок
```

## Деплой на Railway

### Шаг 1 — Подготовь репозиторий

Загрузи все файлы из `bot_v3/` в **корень** репозитория `Ads_TGbot`:
- `bot.py`
- `database.py`
- `character.py`
- `keyboards.py`
- `requirements.txt`
- `Procfile`

### Шаг 2 — Создай Railway-проект

1. Зайди на [railway.app](https://railway.app) → New Project → Deploy from GitHub Repo
2. Выбери репозиторий `Ads_TGbot`
3. Railway автоматически определит `Procfile` и запустит `python bot.py`

### Шаг 3 — Добавь Volume (постоянное хранение)

Railway сбрасывает диск при каждом перезапуске, поэтому база данных нужна на постоянном разделе:

1. В проекте Railway → **Volumes** → **Add Volume**
2. Mount Path: `/data`
3. После этого Railway перезапустит сервис

### Шаг 4 — Задай переменные окружения

Railway → Variables → добавь:

```
TG_TOKEN         = <токен от @BotFather>
OPENROUTER_API_KEY = <ключ с openrouter.ai>
LLM_MODEL        = openrouter/owl-alpha
DB_PATH          = /data/dnd_bot_v3.db
```

### Шаг 5 — Деплой

Сделай push в GitHub → Railway автоматически передеплоится. Статус проверяй в разделе **Deployments**.

## Одиночная игра

Одиночная игра работает в **личных сообщениях** с ботом.

### Сценарий первого запуска

```
/start
→ Визард создания персонажа:
  1. Выбор расы (8 рас с кнопками)
  2. Выбор класса (8 классов с кнопками)
  3. Характеристики (бросок 4d6 или стандартный набор)
  4. Ввод имени
  5. Подтверждение → DM начинает историю

→ Под каждым ответом DM — кнопки действий:
  🎲 Бросить d20  ⚔️ Атака  📜 Мой лист
  💤 Короткий отдых  🌙 Долгий отдых
  💾 Сохранить  📖 Помощь
```

### Как общаться с DM

Просто пиши что делаешь — DM ответит:
```
Игрок: Я осторожно подхожу к двери таверны и прислушиваюсь
DM: [описывает сцену, возможно бросает Восприятие...]

Игрок: Достаю меч и атакую гоблина!
DM: [описывает бой, автоматически обновляет HP...]
```

## Групповая игра

Групповая игра работает в **Telegram-группе** с 2–10 игроками.

### Настройка группы

```
1. Добавь бота в группу (нужны права отвечать на сообщения)
2. Каждый игрок в личке с ботом: /start → создай персонажа
3. В группе каждый пишет: /join
4. Проверь состав: /party
5. Начинай приключение — просто пиши!
```

### Порядок игрового процесса

```
Игрок 1: Предлагаю войти в лес и поискать следы
DM: [описывает лес, все персонажи учтены в контексте]

Игрок 2: Мой персонаж Таша бросает Природу
→ /check природа
DM: Результат — ты замечаешь свежие следы орка...

/fight Орк   ← кто-то начинает бой
→ 🎲 Инициатива: Арес 18, Таша 14, Орк 9
→ ⚔️ Первый ход: @ares_player!

Игрок 1 (Арес): атакую орка
→ /done    ← передаю ход

⚔️ Теперь ход: @tasha_player!
Игрок 2 (Таша): использую заклинание
→ /spell 1 Магическая стрела → /done

[... DM описывает действие орка ...]
```

### Важные детали групповой игры

- **История общая** — все игроки видят одно повествование DM
- **Персонажи раздельные** — у каждого свой лист, HP, инвентарь
- **Квесты общие** — добавляются для всей группы
- **В свой ход** — только текущий игрок может действовать (если идёт бой)
- **Вне боя** — любой игрок может писать DM в любое время

## Авто-трекинг сюжета

Это ключевая функция v3. DM автоматически замечает что происходит в сцене и обновляет лист персонажа без ручных команд.

### Как это работает

1. Игрок пишет: *«Побеждаю гоблина и обыскиваю его тело»*
2. DM рассказывает историю, а в конце ответа добавляет скрытый блок:
   ```
   [STATE]{"hp_change":-5,"gold":20,"exp":50,"items_add":["Короткий меч"],"items_remove":[],"abilities_add":[],"quests_add":[],"quests_done":[]}[/STATE]
   ```
3. Бот парсит блок, применяет изменения и показывает итоги:
   ```
   ━━━━━━━━━━
   🩸 -5 HP → 45/50
   💰 +20 зм → 70 зм
   ⭐ +50 XP → 350/900
   🎁 Получено: Короткий меч
   ```
4. Текст `[STATE]...[/STATE]` скрывается — игрок видит только нарратив

### Что DM отслеживает автоматически

| Событие | Поле STATE |
|---------|-----------|
| Урон от врага | `hp_change: -N` |
| Лечение | `hp_change: +N` |
| Найденное золото | `gold: +N` |
| Потраченное золото | `gold: -N` |
| Опыт за победу/событие | `exp: N` |
| Найденный/купленный предмет | `items_add: ["Предмет"]` |
| Использованный/проданный предмет | `items_remove: ["Предмет"]` |
| Новая способность или заклинание | `abilities_add: ["Название"]` |
| Новый квест | `quests_add: ["Название квеста"]` |
| Выполненный квест | `quests_done: [ID]` |

### Автоматическое повышение уровня

Когда накоплено достаточно XP, бот автоматически:
- Увеличивает уровень
- Прибавляет HP (хит-кубик + мод. Телосложения)
- Обновляет слоты заклинаний (для кастеров)
- Показывает уведомление: `🎉 УРОВЕНЬ 5! +7 HP макс.`

## Система ходов и инициатива

### Запуск боя

```
/fight Дракон-дракон
→ Если в группе > 1 игроков — автоматически бросается инициатива:
  🎲 Инициатива:
  1. @artur: 15 +2 = 17
  2. @olga: 12 +3 = 15
  3. Дракон: 11
  
  ⚔️ Первым ходит: @artur!
```

### Порядок хода

- Бот анонсирует чей ход: `⚔️ Раунд 2 — ход: @username`
- Текущий игрок действует (пишет, жмёт кнопки, использует команды)
- Игрок пишет `/done` → ход переходит следующему
- Если кто-то действует не в свой ход — бот предупреждает

### Разрешение ничьей

При одинаковой инициативе порядок определяется случайно (равновероятно).

### Завершение боя

```
/end_combat
→ Сбрасывается порядок ходов
→ Убирается боевое состояние у всех игроков
→ Игра продолжается в режиме свободного повествования
```

## Приватные листы персонажей

### В личном чате

`/info` или кнопка «📜 Мой лист» — показывает полный лист только тебе.

### В групповом чате

`/info` — бот **отправляет лист в личные сообщения**, в группу пишет только:
```
📜 Арес, лист отправлен тебе в личку!
```

Другие игроки не видят твои характеристики, инвентарь, золото и слоты заклинаний.

### Публичный статус партии

`/party` — показывает всем только имя, расу, класс, уровень и полоску HP каждого игрока:
```
👥 Состав партии:
• Арес (@artur)  Полуорк Варвар ур.3  ❤️42/60 ██████░░
• Таша (@olga)   Эльф Маг ур.3        ❤️18/24 ███░░░░░
```

### Настройка получения личных сообщений

Чтобы бот мог отправить лист в личку, игрок должен **хотя бы раз написать `/start` боту в личных сообщениях**. Если этого не сделано, бот покажет краткий вариант листа прямо в группе.

## Боевая система

### Начало боя

```
/fight Орк-воин
→ Враг создаётся автоматически:
  • HP: случайное число (10–35)
  • КД: случайное (10–16)
  • XP за победу: случайное (25–100)
```

### Атака через кнопку ⚔️

1. Бросок `d20 + макс(мод.Силы, мод.Ловкости)` vs КД врага
2. Если попадание: `d8 + мод.Силы` урона врагу
3. Если критическое попадание (20): урон × 2
4. Если промах или критический промах (1): враг контратакует
5. После хода игрока — враг всегда контратакует (3–10 урона)
6. При HP врага ≤ 0 — победа, DM нарративно описывает исход

### Кнопки в бою

| Кнопка | Эффект |
|--------|--------|
| ⚔️ Атаковать | Бросок атаки vs КД врага |
| 🔮 Заклинание | Напоминает команду `/spell` |
| 🛡️ Уклонение | Нарративное уклонение (DM даёт помеху) |
| 💨 Рывок | DM нарративно описывает перемещение |
| 🏃 Бежать | Бросок `d20 + мод.Ловкости` vs 12 |
| ✅ Завершить ход | `/done` в группе |

### Побег

```
Бросок ловкости: d20 + мод.Ловкости vs 12
≥ 12 → успех, бой завершён
< 12 → провал, враг атакует в спину (4–12 урона)
```

### Методы создания характеристик

**Бросок 4d6** — бросается 4 кубика d6, наименьший отбрасывается, сумма = характеристика. Повторяется 6 раз, сортируется по убыванию и распределяется.

**Стандартный набор** — 15, 14, 13, 12, 10, 8 в порядке Сила → Ловкость → Телосложение → Интеллект → Мудрость → Харизма. Предсказуемо и честно для всех.

## База данных

Используется SQLite. При деплое на Railway необходим Volume для постоянного хранения.

### Таблицы

#### `players` — персонажи
```sql
(group_id, user_id)   -- уникальный ключ, group_id = chat_id группы
name, race, class      -- основные данные
level, exp, exp_next   -- прогресс
hp, max_hp, armor_class, hit_die  -- боевые характеристики
strength/dexterity/constitution/intelligence/wisdom/charisma  -- характеристики
inventory, abilities   -- JSON-массивы предметов и способностей
spell_slots            -- JSON {уровень: [текущие, макс]}
combat_state           -- JSON с состоянием текущего боя (или NULL)
gold                   -- золото
```

#### `history` — история разговора
```sql
group_id    -- чья история (одна на группу)
role        -- 'user' | 'assistant' | 'system'
content     -- текст сообщения
user_id     -- кто писал (для группы)
is_summary  -- 1 если это сжатое изложение
```

#### `quests` — квесты
```sql
group_id    -- к какой группе относится
title       -- название квеста
status      -- 'active' | 'completed'
```

#### `party` — участники группы
```sql
group_id, user_id, username  -- уникальный ключ
initiative  -- последний бросок инициативы
```

#### `turn_state` — состояние ходов в группе
```sql
group_id       -- ключ
turn_order     -- JSON-массив user_id в порядке инициативы
current_idx    -- индекс текущего игрока
round_number   -- номер раунда
mode           -- 'combat' | 'exploration'
```

#### `saved_games` — сохранения
```sql
group_id, user_id, save_name  -- уникальный ключ
player_data, history_data, quest_data  -- JSON-снимки состояния
```

### Ключ персонажа: group_id + user_id

В личных чатах `group_id == user_id` (Telegram присваивает личному чату ID равный ID пользователя).

В групповых чатах `group_id = ID группы` (отрицательное число), `user_id = ID игрока`.

Это позволяет одному игроку иметь **разных персонажей** в разных группах.

## Сжатие истории

Telegram-боты ограничены размером контекста AI-модели. При длинных приключениях история автоматически сжимается.

### Параметры (в `bot.py`)

```python
COMPRESS_AT  = 40   # сжимать когда история > 40 сообщений
KEEP_RECENT  = 20   # всегда хранить последние 20 сообщений свежими
MAX_HISTORY  = 30   # отправлять в AI последние 30 сообщений
```

### Как работает сжатие

1. Накопилось > 40 сообщений
2. Берётся (40 - 20) = 20 самых старых сообщений
3. AI получает задание: *«Кратко (4–6 предложений) перескажи события»*
4. 20 старых сообщений удаляются
5. Вместо них в базу записывается одно системное сообщение: `📜 [Краткое изложение]...`
6. Этот свиток памяти передаётся DM вместе с остальной историей

Таким образом DM всегда «помнит» что было раньше, но контекст не переполняется.

## Частые вопросы

**Q: Бот не отвечает в группе**  
A: Убедись, что у бота есть права читать и отправлять сообщения. В настройках группы (Privacy Mode) — для чтения всех сообщений нужно выключить Privacy Mode через @BotFather командой `/setprivacy → Disable`.

**Q: /info не отправляет лист в личку**  
A: Пользователь должен сначала написать `/start` боту в личных сообщениях. После этого бот сможет писать ему напрямую.

**Q: База данных сбрасывается при рестарте Railway**  
A: Добавь Railway Volume с mount path `/data` и задай `DB_PATH=/data/dnd_bot_v3.db`.

**Q: AI не отвечает / timeout**  
A: Проверь `OPENROUTER_API_KEY`. Бот делает 3 попытки с паузой, после чего сообщает об ошибке.

**Q: DM не обновляет лист автоматически**  
A: Авто-трекинг работает через [STATE] блок в ответе DM. Некоторые запросы DM не считает нужным сопровождать изменениями (обычный диалог без событий) — это нормально. При боях, находках и важных событиях блок появляется всегда.



Что стоит добавить:

Команда /бой — выводит текущее состояние боя: чей ход, раунд, HP всей партии. Чтобы войдя через сутки игрок сразу видел картину без скроллинга.
Автопередача по желанию — добавить в системный промпт фразу вроде "напиши [DONE] если ход закончен", чтобы AI сам сигнализировал когда нужен следующий игрок.
Подтверждение уточнения — если нет реплая на DM, а игрок пишет обычный текст, бот может его поймать: "Это ответ на вопрос DM? Напиши /да чтобы подтвердить".
История раунда — в конце каждого раунда бот суммирует что сделал каждый игрок. Полезно после перерыва.
Итого: система рабочая и реально удобнее для асинхронной игры чем большинство готовых ботов. Главный риск — потеря контекста при длинных перерывах. Команда /бой для дайджеста состояния — это первое что стоит добавить.
